Delivery Platform Enterprise Solution


Our team came up with an Enterprise Solution (Delivery Platform) 📦 concept and developed it for a coursework (IS213 Enterprise Solution Development) in April 2021. The Enterprise Solution is built on the Microservices Architecture and focuses on server-side architectural design. The design emphasizes on developing loosely-coupled and reusable simple services, and leverages on composite services as the orchestrator to invoke sequences of events.

Overview

The delivery platform allows users to obtain on-demand delivery services that is supported by a group of zealous delivery drivers.
Depending on the role or account type, customers and drivers will have access to different functionalities and user interfaces. For instance, Customers will be able to view their delivery orders while Drivers will be able to view the Schedules.

User Scenarios

  • 01 User SignUp and Login
  • 02 Customer creates New Delivery Order
  • 03 Customer makes Payment
  • 04 Driver views all or specific Schedule
  • 05 Driver Updates a Delivery
  • 06 Customer Views Delivery Details and Status

Technicalities

    Microservices (MS)

  • Atomic MS:
  • • Login, Customer, Driver, Delivery, Schedule, Payment
  • Composite MS:
  • • Orchestration Pattern
  • • RegisterUser, RegisterTelegramNotification (TelegramBot), ProcessOrder, ScheduleDriver, CustomerViewDetails, DriverViewDetails, CompleteDelivery
  • • RegisterTelegramNotification uses the TelegramBot platform to automate Notification Registration
  • Subscribers:
  • • CustomerNotification, DriverNotification, Error

    • Communication Protocol

  • • Synchronous HTTP RESTful APIs (Flask & TelegramBot)
  • • Asynchronous Message-based User Notifications (RabbitMQ, AMQP)

    • External APIs

  • • Stripe API for Payment, Google Matrix Distance for Delivery Distances and Time, Telegram API for Notifications

    • Other Technologies

  • • Deployed to Docker containers using Docker Compose for each Services
  • • Kong API Gateway

    • Databases

  • • Database per Service Pattern
  • • AWS RDS (mySQL)
  • • PostgresSQL to store Kong configurations

Takeaways

It was a really fun and enriching experience and I learnt immensely from this coursework in designing Microservices architecture that focuses on modularity and reusability, developing a Full-Stack Enterprise Solution and experiencing how the different components of the IT solution complement one another.

I had the opportunity to explore various technologies that were used for different purposes and utilized external API calls such as Stripe payment service to enhance the functionality of our IT solution. Additionally, I had the opportunity to experience building Back-End processes such as HTTP REST APIs, messaging-based AMQP services and Telegram Chatbot as a composite microservice. Lastly, I had the chance to employ deployment tools such as Docker, Docker Compose and middleware such as Kong API Gateway.

This project was such a fulfilling one and it widened my perception of Enterprise Solutions concepts, design patterns, interactions between Microservices and integration between client-side interfaces and downstream services. Looking forward to future projects!

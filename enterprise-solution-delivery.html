Delivery Platform Enterprise Solution


Our team came up with an Enterprise Solution (Delivery Platform) ideation and developed it for a coursework in April 2021.

The Enterprise Solution is built on the Microservices Architecture and focuses on server-side architectural design. The design emphasizes on developing loosely-coupled and reusable simple services, and leverages on composite services as the orchestrator to invoke sequences of events.

Overview

The delivery platform allows users to obtain on-demand delivery services, supported by a group of delivery drivers.
Depending on the role or account type, customers and drivers will have access to different functionalities and user interfaces. For instance, Customers will be able to view their delivery orders while Drivers will be able to view the Schedules.

User Scenarios

  • 01 User SignUp and Login
  • 02 Customer creates New Delivery Order
  • 03 Customer makes Payment
  • 04 Driver views all or specific Schedule
  • 05 Driver Updates a Delivery
  • 06 Customer Views Delivery Details and Status

Technicalities

    Services

  • • Microservices Architecture (9 Simple 7 Composite Microservices)
  • • Telegram Chatbot as a Composite Microservice to automate Notification Registration

    • Communication Protocol

  • • Synchronous Communication via HTTP RESTful APIs (Flask & Telegram Bot)
  • • Asynchronous Communication via AMQP for User Notifications (RabbitMQ)
  • • Topic Exchange to direct messages to Customer, Driver and Error Subscribers

    • External APIs

  • • Stripe API for payment, Google Matrix Distance for Delivery Distances and Time, Telegram API for Notifications

    • Other Technologies

  • • Deployed to Docker containers using Docker Compose for each Services
  • • API Gateway using Kong and Konga

    • Databases

  • • PostgresSQL to store Kong configurations
  • • Database per Service Pattern
  • • AWS RDS (mySQL)

Takeaways

I learnt immensely from this coursework in terms of designing a loosely coupled solution architecture, developing a Full-Stack Enterprise Solution as well as experiencing how everything comes together.
I had the opportunity to explore the various technologies used for different purposes and utilise external API calls such as Stripe to enhance the functionality of our IT solution. Additionally, I had the opportunity to experience building Back-End processes such as synchronous HTTP REST APIs, asynchronous messaging-based AMQP services and Telegram Chatbot as an orchestrator. Lastly, I had fun utilizing deployment tools such as Docker, Docker Compose and Kong API Gateway.

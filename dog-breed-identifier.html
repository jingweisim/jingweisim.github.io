Telegram ChatBot Dog Breed Identifier


Hack&Roll 2021 was a 24 hours Hackathon that spanned from 8 to 9 Jan 2021.

As a fun side project to explore Machine Learning and Software Development, our team decided to develop a Telegram Chatbot and host a Dog Breed Classification model as a means for learning and growth.

A quick run through of the Bot:

Features

The Telegram Chatbot allows users to submit an image of a dog and utilises a Convolutional Neural Network model to classify the breed class. If users were to send a message to the Telegram Bot, the Bot will reply with an image or gif of a dog while it awaits an dog image from the user.

Technicalities

  • • Telegram Bot API
  • • CNN Model using TensorFlow
  • • Transfer Learning using ImageNet
  • • 99% Training Accuracy and 84% Test Accuracy
  • • Deployed on Heroku
  • • Ideation to Model Training to Deployment during Hackathon

Process and Takeaways

Throughout this valuable learning experience, we had the opportunity to source for a suitable dataset for our Machine Learning model, perform Exploratory Data Analysis to identify any missing data and execute data imputation if necessary. We decided on the Stanford Dogs Dataset as it featured 20,000 images of 128 classes of dog breeds that was sufficient for model training and contained a diverse selection of dog breeds.

We performed data preparation on the image dataset, such as image resizing, image transformation into numpy arrays and utilising TensorFlow documentation for data input pipeline building. Additionally, we underwent selection of the model, model building, model training and evaluation through performance testing. Through the model selection phase, we selected ImageNet as it was previously trained on similar datasets. We utilized a headless model and applied a dense layers to classifying the image into the 128 dog breed classes.

During model training, we freeze the headless model while allowing the dense layer to train and undergo weight adjustments accordingly. Despite the high training accuracy of 99% on 10 epochs, there was a drop in the validation accuracy (84%) which could be showing signs of overfitting of the training data.

It was an enriching experience that opened my perception to the possibilities and capabilities of Deep Learning and Computer Vision.
